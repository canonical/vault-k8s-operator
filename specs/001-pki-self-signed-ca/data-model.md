# Data Model: PKI Self-Signed CA

**Feature**: PKI Self-Signed CA  
**Date**: 2026-05-25  
**Spec**: [spec.md](spec.md) | **Plan**: [plan.md](plan.md)

## Entities

### SelfSignedCA

The self-signed CA certificate and its associated private key, generated by Vault and managed by the charm.

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| certificate | str (PEM) | Vault `pki/root/generate/internal` | The self-signed CA certificate |
| private_key | str (PEM) | Vault `pki/root/generate/internal` | The CA private key |
| common_name | str | Juju config `pki_ca_common_name` | The CN used when generating the CA |
| sans_dns | list[str] | Juju config `pki_ca_sans_dns` | Subject alternative DNS names |
| ttl | str | Hardcoded default or config | Validity period (e.g., "87600h" for 10 years) |
| country | str \| None | Juju config `pki_ca_country_name` | Country name |
| province | str \| None | Juju config `pki_ca_state_or_province_name` | Province/state name |
| locality | str \| None | Juju config `pki_ca_locality_name` | Locality name |
| organization | str \| None | Juju config `pki_ca_organization` | Organization name |
| organizational_unit | str \| None | Juju config `pki_ca_organizational_unit` | Organizational unit name |

**Storage**: The certificate and private key are stored together in a Juju application secret with label `vault-pki-self-signed-ca`. The remaining fields are derived from charm config at runtime.

**Uniqueness**: One self-signed CA per Vault application deployment. A new CA is generated when `pki_ca_common_name` changes.

**Lifecycle**:
1. **Created**: When `pki_ca_common_name` is set, no external CA relation exists, and Vault is ready
2. **Stored**: Certificate + private key written to Juju secret
3. **Imported**: Imported into Vault PKI mount as intermediate CA issuer
4. **Rotated**: When `pki_ca_common_name` changes, a new CA is generated, imported as a new issuer, and made default
5. **Retired**: Old issuer remains in Vault but is no longer the default

### PKIIssuer

An issuer entity within the Vault PKI secrets engine.

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| issuer_ref | str | Vault | Unique reference within the mount (e.g., "issuer-1") |
| certificate | str (PEM) | Vault | The CA certificate for this issuer |
| default | bool | Vault config | Whether this issuer is the default for signing |

**Storage**: Stored within Vault's PKI mount (`charm-pki`).

**Relationship to SelfSignedCA**: Each `SelfSignedCA` corresponds to one `PKIIssuer` after import. When rotated, a new `SelfSignedCA` creates a new `PKIIssuer`.

## Validation Rules

1. `common_name` must be non-empty and valid (existing `common_name_config_is_valid()`)
2. `sans_dns` must be a valid comma-separated list (existing `sans_dns_config_is_valid()`)
3. `ttl` must be a valid Vault TTL string (default: "87600h" = 10 years)
4. The CA certificate validity must be greater than the role's max TTL (enforced by Vault)
5. Leaf certificate validity is half the CA validity (existing `_PKIUtils.calculate_certificates_ttl()`)
